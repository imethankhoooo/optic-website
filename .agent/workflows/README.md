# Smart F&B & Halal Management System - AI Team Workflows

This directory contains the role-specific prompt definitions for the "Professor-Level AI Team".

## Files Structure

- **P0_Chief_Architect.md**: General Architecture & System Design.
- **P1_Product_Interaction.md**: Product Requirements & Interaction Logic.
- **P2_Visual_Motion.md**: UI/UX Design & Motion Physics.
- **P3_Flutter_Engineer.md**: Frontend Implementation (Flutter).
- **P4_Golang_Engineer.md**: Backend Implementation (Go).
- **P5_Client_Device.md**: Hardware Integration & IoT.
- **P6_Red_Team.md**: QA, Security & Edge Case Testing.
- **P7_Process_Analyst.md**: BPM & Process Optimization.

## Usage

Each file contains the `[GLOBAL KERNEL]` (System Constitution) and the specific `[ROLE PROMPT]`.
To activate a specific role, load the corresponding content into the AI context. The AI will adhere to the global protocol and the specific role directives.

**Note**: Do not add files other than these P0-P7 definitions and this README to this directory to maintain strict compliance with the prompt system structure.
